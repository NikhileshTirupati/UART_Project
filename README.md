A fully parameterized, Verilog-based **Universal Asynchronous Receiver/Transmitter (UART)** implementation featuring a baud rate generator, transmitter, and receiver — all integrated into a single top-level module. The design is simulation-ready with individual and integrated testbenches.

## Overview

This project implements the UART serial communication protocol in Verilog. UART is an asynchronous protocol that transmits data serially, one bit at a time, with a configurable baud rate. The design supports:

- Configurable system clock frequency (`XTAL`)
- Configurable baud rate (`BAUD`)
- Configurable data width (`WIDTH`, default 8 bits)
- Standard UART framing: **1 start bit + N data bits + 1 stop bit**
- Active-low asynchronous reset
- Loopback-capable architecture (TX output can be wired to RX input)

## Module Descriptions

### `uart` (Top-Level)

**File:** `src/design/uart.v`

The top-level wrapper that instantiates and connects the three submodules: `u_baud`, `u_xmit`, and `u_rec`. The baud clock generated by `u_baud` drives both the transmitter and receiver. The module exposes all control and data signals to the outside.

```
sys_clk ──► u_baud ──► uart_clk ──► u_xmit ──► uart_XMIT_dataH
                                 └──► u_rec  ◄── uart_REC_dataH
```

---

### `u_baud` (Baud Rate Generator)

**File:** `src/design/u_baud.v`

Generates the internal UART clock by dividing the system clock. The division ratio is computed as:

```
CLK_DIV = XTAL / (BAUD × 16 × 2)
```

This produces an oversampled clock at **16× the baud rate**, which the transmitter and receiver use for precise bit timing and mid-bit sampling.

---

### `u_xmit` (Transmitter)

**File:** `src/design/u_xmit.v`

A 3-state FSM-based UART transmitter. It serializes parallel input data and sends it over the `uart_XMIT_dataH` line with proper UART framing.

**Frame format (LSB first):**

```
[START=0] [D0 D1 D2 D3 D4 D5 D6 D7] [STOP=1]
```

**States:**

| State   | Description                                   |
| ------- | --------------------------------------------- |
| `IDLE`  | Line held high; waits for `xmitH` to go high  |
| `TXMIT` | Shifts out bits one at a time every 16 clocks |
| `DONE`  | Transmission complete; asserts `xmit_doneH`   |

---

### `u_rec` (Receiver)

**File:** `src/design/u_rec.v`

A 3-state FSM-based UART receiver. It samples the incoming serial line, detects the start bit, and reconstructs the parallel data word.

**States:**

| State  | Description                                                                |
| ------ | -------------------------------------------------------------------------- |
| `IDLE` | Waits for a falling edge (start bit) on `uart_REC_dataH`                   |
| `REC`  | Samples each data bit at the center (mid-bit sampling at counter == 15/16) |
| `DONE` | Validates the stop bit; outputs `rec_dataH` and asserts `rec_readyH`       |

Mid-bit sampling is used to improve noise immunity — the receiver waits for ~half a bit period before sampling.

## Port Reference

### Top-level `uart` module

| Port              | Direction | Width   | Description                        |
| ----------------- | --------- | ------- | ---------------------------------- |
| `sys_clk`         | Input     | 1       | System clock                       |
| `sys_rst_l`       | Input     | 1       | Active-low asynchronous reset      |
| `xmit_H`          | Input     | 1       | Transmit enable (high to send)     |
| `xmit_dataH`      | Input     | `WIDTH` | Parallel data to transmit          |
| `uart_REC_dataH`  | Input     | 1       | UART serial receive line (RX)      |
| `uart_XMIT_dataH` | Output    | 1       | UART serial transmit line (TX)     |
| `xmit_doneH`      | Output    | 1       | High when transmission is complete |
| `rec_dataH`       | Output    | `WIDTH` | Received parallel data             |
| `rec_readyH`      | Output    | 1       | High when received data is valid   |
| `rec_busy`        | Output    | 1       | High while receiver is active      |
| `xmit_active`     | Output    | 1       | High while transmitter is sending  |

## FSM Design

Both `u_xmit` and `u_rec` use a standard two-always-block FSM pattern:

- **Sequential block:** Updates current state (`cs`) and registered outputs on the clock edge (or reset).
- **Combinational block:** Computes next state (`ns`) and combinational outputs based on current state and inputs.

This separation ensures clean timing and avoids latches.

### Loopback Test (`uart_tb.v`)

The top-level testbench connects `uart_XMIT_dataH` directly to `uart_REC_dataH`, enabling a loopback test. The transmitter sends a byte and the receiver captures it on the same DUT.

```
uart_XMIT_dataH ──┐
                  └──► uart_REC_dataH  (loopback)
```

**Test sequence:**

1. Apply reset, then deassert.
2. Transmit `0xA6`.
3. Wait for `rec_readyH` to confirm reception.
4. Transmit `0xAA`.
